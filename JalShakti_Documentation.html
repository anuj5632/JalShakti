📋 Problem Statement

💬 "Bad water for weeks, nobody listens"

- Common complaint from rural and urban communities across India

🔍 The Challenge

Millions of Indians lack access to safe drinking water. The problems include:

  • No Real-time Monitoring: Water quality is tested infrequently, if at all
  • Complex Reporting: Filing complaints to government is difficult and time-consuming
  • Language Barriers: Technical information isn't accessible in local languages
  • Delayed Response: By the time issues are reported, damage is already done
  • Lack of Awareness: People don't understand what makes water unsafe

✨ Our Solution: JalShakti

An end-to-end IoT platform that monitors water quality in real-time, predicts issues before they occur, automatically alerts users, and files government complaints - all in the user's preferred language.

🎯 Target Users

🏘️ Rural Communities

Village water tanks, hand pumps, community wells with multilingual support

🏢 Urban Societies

Apartment complexes, housing societies, commercial buildings

🏛️ Government Bodies

Municipal corporations, Jal Jeevan Mission, PHE departments

🏥 Institutions

Schools, hospitals, hostels requiring safe water supply

✨ Key Features

📊 1. Real-Time Dashboard

Live sensor readings updated every 3 seconds with beautiful visualizations

  • Quality Score: Calculated using WHO standards with weighted formula
  • Trend Graphs: 24-hour history for all parameters
  • Color Coding: Green (Safe) → Yellow (Warning) → Red (Critical)
  • Individual Scores: pH Score, TDS Score, Turbidity Score breakdown
// Quality Score Formula (WHO Standards) Quality Score = (pH_Score × 0.30) + (TDS_Score × 0.30) + (Turbidity_Score × 0.40) // Thresholds: Safe: Quality ≥ 70% → Green Warning: Quality 50-70% → Yellow Critical: Quality < 50% → Red

🔔 2. Smart Alarm System

⚠️ Warning Alarm

Triggers at quality < 70%. Moderate beeping pattern to alert users.

🚨 Critical Alarm

Triggers at quality < 50%. Urgent, rapid alarm requiring immediate attention.

  • Uses Web Audio API - works in all modern browsers
  • Mute toggle for user control
  • Different patterns distinguish severity levels
  • 30-second debounce prevents alarm fatigue

📧 3. Automatic Email Alerts

Zero configuration required! Uses the email from Google login automatically.

  • Sends to registered Google email - no manual entry needed
  • Beautiful HTML emails with all sensor readings
  • Toggle on/off as per user preference
  • 30-second cooldown between alerts (demo-friendly)
  • Includes quality score, all sensor values, and recommended actions

🚀 How to Run

📋 Prerequisites

  • Python 3.10+
  • Node.js 16+
  • npm or yarn
  • Modern web browser

⚙️ Backend Setup

# Navigate to backend folder cd backend # Install Python dependencies pip install -r requirements.txt # Start the server python -m uvicorn main:app --port 8000 # Server runs at http://localhost:8000 # API docs at http://localhost:8000/docs

🎨 Frontend Setup

# Navigate to frontend folder cd frontend # Install Node dependencies npm install # Start development server npm start # Opens at http://localhost:3000

🎮 Demo Presentation Steps

  1. Start both backend and frontend
  2. Open http://localhost:3000
  3. Login with Google account
  4. Press 1 for Normal readings (Green dashboard)
  5. Press 2 for Warning scenario (Alarm triggers)
  6. Click File Complaint button (appears automatically)
  7. Select Hindi/Marathi language and submit
  8. Click 💬 Chatbot button
  9. Try quick replies in different languages
  10. Press 3 for Critical scenario
  11. Show email received in inbox
